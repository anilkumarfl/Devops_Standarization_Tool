# AWS Infrastructure Naming Conventions

All AWS resources generated by the DevOps Standardization Tool must strictly adhere to the following naming pattern to ensure predictability, grouping, and adherence to AWS character limits.

## 1. Naming Pattern
**Pattern:** `{project}-{env}-{resource-type}`

### Constraints
* **Format:** All lowercase, alphanumeric, separated by hyphens only (no spaces, no underscores, no camelCase).
* **Environment (`{env}`):** Valid values are `dev`, `staging`, and `prod`.
* **Project Name (`{project}`):** Extracted from `infra.yaml`. **Must not exceed 20 characters** to safely accommodate tight AWS limits (like the 32-character limit on ALBs).

## 3. Validation Example
AWS enforces a **32-character limit** on Application Load Balancers (ALBs). 
If a project is named `customer-onboarding` (19 chars) and deployed to `staging` (7 chars):
* Generated ALB: `customer-onboarding-staging-alb`
* Total Length: 19 + 1 + 7 + 1 + 3 = **31 characters** (Passes validation).
